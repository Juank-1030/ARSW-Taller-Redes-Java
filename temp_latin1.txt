2. Conceptos bīasicos de redes
Los programas que se comunican a travīes de internet utilizan generalmente dos protocolos: el Transmission Control Protocol (TCP) o el User Datagram Protocol (UDP). En java en general usted utiliza clases ya implementadas en el paquete java.net.
2.1. TCP
El Transmission Control Protocol (TCP) es un protocolo basado en conexiīon que provee una conexi īon confiable entre dos computadores. TCP en particular mantiene el orden de los paquetes de datos y garantiza que todos los datos se entreguen.
2.2. UDP
El User Datagram Protocol (UDP) es un protocolo que envīia los datos en paquetes llamados datagramas, no provee garantīia de entrega ni de orden de entrega. Este protocolo no esta basado en conexiīon.
2.3. Que son los puertos
En general los computadores tiene una sola conexiīon a internet y todos los datos que llegan y salen utilizan esta conexiīon fīisica. Sin embargo, el computador
1

puede tener muī ltiples aplicativos que utilizan la red. Para separar la informaciīon que es enviada a una aplicativo especīifico se asigno un nuī mero līo gico a cada aplicaciīon. Este nuīmero es denominado el puerto y , como veremos m īa s adelante, es utilizado para enviar datos a aplicaciones especīificas en computadores remotos.
Los protocolos de TCP y UDP utilizan estos puertos para enviar los datos que llegan a las aplicaciones correctas. Recuerde que cada aplicaciīon que espera datos de la red se le asigna un puerto para que puede escuchar los datos que llegan a un puerto determinado.
Los puertos se representan con un entero de 16 bits y tienen un rango de 0 hasta 65.535. Los puertos de 0- 1023 est īan restringidos para aplicaciones especīificas por ejemplo el 80 es para el servidor web.

El mecanismo tiene un funcionamiento bīasico en el cual el cliente invoca un mīeto do en el stub (que es un objeto local), y es este el encargado de hacer la invocaciīon del mīeto do en el objeto remoto,
